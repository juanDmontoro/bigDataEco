Introduction

Some General Considerations

 

  • In classification problems, we follow the same scheme for model evaluation and selection as in regression: sample partitioning and resampling.

    • Choosing hyperparameters or tuning parameters via cross-validation allows us to select the optimal specification within a model.
    • Evaluating between models involves assessment outside the training sample.

  • However, classifier accuracy metrics differ from those used in regression problems.

  • All classifier performance metrics can be applied to both training and test data: generally, metrics on training data are expected to be better than those on test data (a model tends to perform better on data it has already seen).

  • Good metrics on the training sample and poor metrics on the test sample are indicative of overfitting: the model ‚Äúlearns‚Äù the training data, including its noise and outliers, which negatively affects its performance on new data (the test sample).

The Null Classifier

 

A null (or naive) classifier is a simple baseline model used in classification tasks that does not use any features to make predictions but relies on the distribution of the target variable in the training data.

Most common types of null classifiers:

  1. Majority class classifier (or most frequent class): always predicts the majority class from the training data.

  2. Random (or stratified) classifier: predicts with probability based on the class distribution in the training data.

The null classifier serves as a benchmark for comparing the performance of more complex models. If a model performs worse than the null classifier, it indicates that it is not effectively capturing patterns in the data.

Classification Errors

 

A binary classifier can make two types of errors:

  1. Type I Error or false positive (FP): predicting a positive outcome (1) for an instance that actually belongs to the negative class (0).
  2. Type II Error or false negative (FN): predicting a negative outcome (0) for an instance that actually belongs to the positive class (1).

 

These errors can be summarized in a confusion matrix.

Confusion Matrix

 

Negative Prediction (0) Positive Prediction (1)
Actual Negative (0) True Negative (TN) False Positive (FP)
Actual Positive (1) False Negative (FN) True Positive (TP)

Accuracy and Precision Metrics

 

Metric Formula Description
Prevalence \[\frac{TP + FN}{TP + TN + FP + FN}\] Proportion of actual positive instances among all instances.
Detection Rate \[\frac{TP}{TP + TN + FP + FN}\] Proportion of true positives among all instances.
Accuracy \[\frac{TP + TN}{TP + TN + FP + FN}\] Proportion of correctly classified instances among all instances.
Precision \[\frac{TP}{TP + FP}\] Proportion of true positives among all instances predicted as positive.
Sensitivity (Recall, TPR) \[\frac{TP}{TP + FN}\] Proportion of true positives among all actual positive instances.
Specificity \[\frac{TN}{TN + FP}\] Proportion of true negatives among all actual negative instances.
False Positive Rate (FPR) \[1-\textrm{Specificity}=\frac{FP}{TN + FP}\] Proportion of false positives among all actual negative instances.
F1 Score \[2 \cdot \frac{\text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}}\] Harmonic mean of precision and recall.
Error Rate \[\frac{FP + FN}{TP + TN + FP + FN}\] Proportion of incorrectly classified instances among all instances.
NIR Proportion of the most frequent class No Information Rate: proportion of the majority class in the dataset.

Log-Loss/Logistic Loss or Cross-Entropy

Log-Loss

 

Metrics derived from the confusion matrix use class predictions, not probabilities. As an alternative, log-loss (also called logistic loss or cross-entropy loss) evaluates the quality of the predicted probabilities produced by a classifier, not just whether the predicted class is correct.

  • It comes from information theory and measures the divergence between the predicted probability distribution and the true distribution (a degenerate 0/1 distribution). The metric quantifies how inefficient or surprising the model‚Äôs predictions are.

  • For binary classification, log-loss is the negative log-likelihood of the Bernoulli model:

    \[ \text{log-loss} = -\frac{1}{n} \sum_{i=1}^n \left[\, y_i \log(\hat{p}_i) + (1 - y_i)\log(1 - \hat{p}_i)\, \right] \]

    where

    • \(y_i \in \{0,1\}\) is the true label,
    • \(\hat{p}_i \in [0,1]\) is the predicted probability that \(y_i = 1\).

A model with well-calibrated probabilities achieves a lower log-loss; confident but incorrect predictions are heavily penalized because \(\log(0)\) tends to \(-\infty\).

Log-Loss

 

  • Log-loss penalizes predictions that are uncertain and especially those that are confidently wrong.
    • When the predicted probability is close to the true label (0 or 1), the loss is low.
    • When the model assigns a low probability to the true class, the loss increases sharply.
  • Two penalty channels
    • Penalty for uncertainty: Correct predictions with low confidence still incur a cost.
    • Penalty for confident mistakes: Assigning high probability to the wrong class produces a very large loss because \(\log(\hat{p})\) approaches \(-\infty\) as \(\hat{p}\) approaches 0.
  • Interpretation
    • Log-loss ranges from 0 (perfect probability predictions) to \(+\infty\) (very confident but wrong predictions).
    • A high log-loss signals substantial problems in the probability estimates‚Äîeither the model is uncertain in general or it is systematically overconfident.
  • Use in learning algorithms. Log-loss is widely used as the cost function in models such as logistic regression, multinomial regression, and many neural networks. During training, minimizing log-loss adjusts the model‚Äôs parameters so that the predicted probabilities better match the empirical data.

Log-Loss: Multinomial Classification

 

The expression generalizes easily to the case of problems with multiple classes, penalizing the low probability assigned to the correct class: \[\textrm{log-loss}=-\frac{1}{n}\sum_i\sum_k(y_{ik}log(\hat{p}_{ik}))\]

ROC Curve

Motivation: ROC curve and AUC

The perfect classifier

  1. Consider a classifier that perfectly discriminates between + and - instances: it assigns to all + instances a predicted probability that is strictly greater than that of all - instances. E.g.:
    \[ y=\{0,0,0,1,1,1\} \quad \longrightarrow \quad \hat{P}(y_i=1\mid X_i)=\{0.05,0.10,0.30,0.35,0.45,0.55\}. \]

  2. Note that in this case the probability that a randomly chosen + instance ranks higher than a randomly chosen - instance is 1.

  3. Now let us manipulate the threshold. If we set the threshold strictly greater than 0.55, no instance is classified as +. In this case TPR = 0 and FPR = 0.

  4. If we set the threshold to exactly 0.35, then TPR = 1 and FPR = 0. All + instances are correctly classified and none of the - instances are misclassified.

  5. Further reducing the threshold induces classification errors:
    ‚Äì at 0.30 ‚Üí (TPR = 1, FPR = 0.33)
    ‚Äì at 0.10 ‚Üí (TPR = 1, FPR = 0.66)
    ‚Äì at 0.05 ‚Üí (TPR = 1, FPR = 1)

  6. The ROC curve of this perfectly discriminating classifier moves vertically along the y‚Äëaxis from (0,0) to (0,1) as the threshold decreases to 0.35 (achieving TPR = 1), and then horizontally along the x‚Äëaxis from (0,1) to (1,1) as the threshold is further reduced below 0.35.

  7. This creates a perfect square under the ROC curve, whose area is equal to 1.

  8. This metric‚Äîthe area under the ROC curve (AUC)‚Äîis a performance measure that combines sensitivity and specificity and quantifies the classifier‚Äôs ability to discriminate between classes.

Motivation: ROC curve and AUC

The pure‚Äëchance classifier

Think of a dumb classifier that assigns a class to each instance by flipping a fair coin. Then

\[ \hat{P}(y=1\mid X)=0.5. \]

The dataset is the same as before:

\[ y=\{0,0,0,1,1,1\}. \]

The ROC curve is built as follows:

  1. For any threshold \(> 0.5\), all units are classified as negative. Then TPR = 0 and FPR = 0.

  2. For any threshold \(\le 0.5\), all units are classified as positive. Then TPR = 1 and FPR = 1.

The ROC is a diagonal: the two points (0,0) and (1,1) define a straight line from the origin to the top‚Äëright corner.

This corresponds to an AUC of 0.5, representing pure‚Äëchance performance.

ROC Curve

 

  • An ROC (Receiver Operating Characteristic) curve is a graphical representation that illustrates the performance of a binary classification system as its discrimination threshold is varied. The ROC curve is created by plotting the true positive rate (TPR) against the false positive rate (FPR) at different threshold settings.

  • Characteristics:

    • True Positive Rate (TPR, also sensitivity or recall): proportion of actual positives correctly identified by the classifier.
    • False Positive Rate (FPR or 1-specificity): proportion of actual negatives incorrectly identified as positive by the classifier.
    • Diagonal line: represents a random classifier with no discriminative ability (i.e., a 50% probability).
    • Area Under the Curve (AUC): the area under the ROC curve (AUC) quantifies the overall ability of the classifier to discriminate between positive and negative classes. An AUC of 1 indicates a perfect classifier, while an AUC of 0.5 indicates a classifier with no discriminative ability (diagonal line).

The ROC curve plots the True Positive Rate (Recall) against the False Positive Rate (1 - Specificity) for different threshold values. The area under the ROC curve (AUC) is a measure of the model‚Äôs ability to distinguish between positive and negative classes.

Code
# Predict the probability of belonging to each class using the fitted model
logit_probs = logistic.predict_proba(X_train)

# Define a function that displays the ROC curve and some threshold values
def plot_roc(labels, predictions, thresholds_every=10, title='',sizex=6,sizey=6):
  # fp: false positive rates. tp: true positive rates
  fp, tp, thresholds = metrics.roc_curve(labels, predictions)
  roc_auc = metrics.auc(fp, tp)

  plt.figure(figsize=(sizex, sizey))
  plt.plot(fp, tp, label='ROC curve (area = %0.2f)' % roc_auc, linewidth=2, color='darkorange')
  plt.plot([0, 1], [0, 1], color='navy', linestyle='--', linewidth=2)
  plt.xlabel('False Positive Rate')
  plt.ylabel('True Positive Rate')
  plt.xlim([-0.03, 1.0])
  plt.ylim([0.0, 1.03])
  plt.title(title)
  plt.legend(loc="lower right")
  plt.grid(True)

  # Show some thresholds
  thresholdsLength = len(thresholds)
  colorMap=plt.get_cmap('jet', thresholdsLength)
  for i in range(0, thresholdsLength, thresholds_every):
    threshold_value_with_max_four_decimals = str(thresholds[i])[:5]
    plt.text(fp[i] - 0.03, tp[i] + 0.005, threshold_value_with_max_four_decimals, fontdict={'size': 15}, color=colorMap(i/thresholdsLength));
  plt.show()

# Apply the function to training data and predictions on training set
plot_roc(y_train,logit_probs[:,1],thresholds_every=1000,
      title='Logistic classifier: training sample')

Uses of the ROC Curve

 

  • Model comparison: ROC curves and AUC are used to compare the performance of different classifiers. AUC provides a metric of the overall performance of a binary classification model by quantifying its ability to distinguish between positive and negative classes. The AUC value ranges between 0 and 1, where:

    • AUC = 1: Perfect classifier
    • AUC = 0.5: Classifier with no discriminative power (equivalent to random guessing)
    • AUC < 0.5: Classifier that performs worse than chance

  • Threshold selection: ROC curves help select the optimal threshold for classification by balancing the trade-off between TPR and FPR.

    • Choose the point on the curve closest (Euclidean distance) to the top-left corner of the plot
    • Evaluate the costs associated with false positives and false negatives. By assigning different costs to these errors, one can determine the threshold that minimizes the total cost.

The AUC represents the probability that a randomly chosen positive instance is ranked above a randomly chosen negative instance.

Class Imbalance

Imbalanced Data in Classification Models

 

Classification models on data with class imbalance often pose additional challenges:

  • Non-informative and unreliable predictive quality metrics:

  • A model can show high accuracy simply by always predicting the majority class.

  • Metrics like accuracy do not reflect true performance on minority classes.

  • Generally, the model ignores the minority class(es):

    • The model tends to ignore or misclassify the less represented classes.
    • Learning is biased as the model captures dominant patterns (ignoring those of minority classes).

  • The model may fail when facing real data where classes are more balanced.

Imbalanced Data in Classification Models: Solutions

 

In general, the existence of imbalanced classes significantly hinders model fitting, requiring additional techniques. Common solutions:

  • Oversampling the minority class
  • Undersampling the majority class
  • Using balanced metrics (unweighted averages over classes or macro avg)
  • Algorithms sensitive to imbalance: e.g., algorithms that train sequentially on errors from previous stages (boosting) or that incorporate an additional cost for errors in the minority class.
