Parametric Models for Categorical Response

Classification Problems

  • Classification problems: qualitative/categorical response
  • Classification techniques
  • Concepts applicable from regression problems: supervised learning, data partitioning, overfitting, underfitting, etc.
  • Linear regression in classification problems?
    • If the response is binary, \(X\hat{\beta}\) is an estimator of \(P(Y=1 \mid X)\)
    • But the predictions \(P(Y \mid X)\) from the linear model will not necessarily fall within the interval \([0,1]\)
    • Impossible to accommodate qualitative responses when we have more than two classes (the response should have an implicit ordering, and even then, the difference between levels would be arbitrary)

Types of Response and Suitable Parametric Models

  • Continuous response: linear regression, Ridge, Lasso
  • Binary response: logistic regression (also probit)
  • Categorical response (more than two options): multinomial regression
  • Count process response (0,1,2â€¦): Poisson regression (or negative binomial if overdispersion)

Linear, logistic, multinomial, or Poisson regression models (among others) are encompassed within the generalized linear model.

Generalized Linear Models (GLMs)

Components of a GLM

  1. A stochastic component specifying the probability distribution of the response variable (EDM, e.g., normal, binomial, Poisson): \[y_i \sim EDM(\mu_i,\sigma^2)\]
  2. A systematic component or linear predictor: \[\nu_i = \beta_1 + \beta_2 x_{2i} + \ldots + \beta_p x_{pi}\]
  3. Link function, i.e., a functional form connecting the mean of the response to the systematic component (the linear predictor): \[g(\mu_i) = \nu_i\]

Since GLMs assume a specific probability distribution for responses from the EDM family, numerical maximum likelihood estimation procedures (e.g., stochastic gradient descent) can be used to estimate parameters. The loss funtion is \[-l(\beta \mid x, y)\]

The Linear Model

  1. Gaussian distribution for the response \(y_i\): \[y_i \sim N(\mu_i, \sigma^2)\]

  2. Linear predictor: \[\nu_i = \beta_0+ \beta_1 x_{i} + \beta_2 x_{2i} + \ldots + \beta_p x_{pi}\]

  3. Identity link function: \[\nu_i = \mu_i\]

The linear model estimates the conditional mean of the response variable as a function of the linear predictor: \[E[y \mid X] = \beta_0 + \beta_1 x_{1i}+ \beta_2 x_{2i} + \ldots + \beta_p x_{pi}\]

Binary Response: The Logistic Model

Let \(y \in {0,1}\) and \(x_1, x_2, \ldots, x_p\) be explanatory variables. In this case:

  1. Stochastic component (Bernoulli distribution): \[P(Y=y_i;\mu_i) = \mu_i^{y_i}(1-\mu_i)^{(1-y_i)}\]

  2. Linear predictor: \[\nu_i = \beta_0 + \beta_1 x_{1i} + \beta_2 x_{2i} + \ldots + \beta_p x_{pi}\]

  3. Link function: \[\nu_i = g(\mu_i) = \log\left(\frac{\mu_i}{1-\mu_i}\right)= \log\left(\frac{P(y=1\mid X)}{P(y=0\mid X)}\right)\]

Combining these elements, the logit model predicts the conditional mean of the response as: \[P(y_i=1 \mid X) = \mu_i = \frac{\exp(\nu_i)}{1+\exp(\nu_i)}\]

Multinomial Logistic Regression

If the response \(y\) has \(k > 2\) classes, we use a multinomial model (also a GLM). Components:

  1. Stochastic component: the response follows a multinomial model: \[P(y_1, y_2, \ldots, y_K) = \frac{n!}{y_1! y_2! \cdots y_K!} p_1^{y_1} p_2^{y_2} \cdots p_K^{y_K}\]

  2. A linear predictor for each category \(j\): \[\nu_j = \beta_{0j} + \beta_{1j} x_1 + \beta_{2j} x_2 + \cdots + \beta_{pj} x_p\]

  3. A softmax link function: \[\log \left(\frac{P(Y = j)}{\sum_k P(Y=k)}\right) = \nu_j\]

Conditional mean: \[P(y=j\mid X)=\frac{\exp(\nu_j)}{\sum_k \exp(\nu_k)}\]

Example: multinomial model

Poisson Model

Count data can be modeled using Poisson specifications (mean equals variance) or Negative Binomial (overdispersion). For Poisson:

  1. Stochastic component: \[P(y_i\mid\lambda) = \frac{\exp(-\lambda)\, \lambda^{y_i}}{y_i!}\]

  2. Linear predictor: \[\nu_i = \beta_{0} + \beta_{1} x_1 + \beta_{2} x_2 + \cdots + \beta_{p} x_p\]

  3. Log link function: \[\log (\lambda) = \nu_i\]

This implies: \[E[y\mid X]=\lambda=\exp(\nu_i)\]

Example: Poisson model

Summary of GLMs

Model Distribution (Stochastic Component) Linear Predictor Link Function
Linear \(y_i \sim N(\mu_i,\sigma^2)\) \(\nu_i = \beta_0 + \beta_1 x_1 + \dots + \beta_p x_p\) Identity: \(\nu_i = \mu_i\)
Logistic (binary) \(y_i \sim \text{Bernoulli}(\mu_i)\) \(\nu_i = \beta_0 + \beta_1 x_1 + \dots + \beta_p x_p\) Logit: \(\log\left(\dfrac{P(Y=1)}{P(Y=0)}\right)=\nu_i\)
Multinomial logistic \(y_i \sim \text{Multinomial}(p_1,\dots,p_k)\) \(\nu_j = \beta_{0j} + \dots + \beta_{pj} x_p\) Logit: \(\log\left(\dfrac{P(Y=j)}{P(Y=k)}\right)=\nu_j\)
Poisson (counts) \(y_i \sim \text{Poisson}(\lambda)\) \(\nu_i = \beta_0 + \dots + \beta_p x_p\) Log: \(\log(\lambda)=\nu_i\)

Regularization in GLMs

Regularized GLMs

  • Regularization is used to avoid overfitting when training GLMs for predictive purposes.
  • Same approach as with linear models: add a regularization term to the loss function penalizing coefficient magnitude.
  • A tuning parameter \(\lambda\) controls the amount of regularization.
  • Optimal \(\lambda\) is chosen via resampling (cross-validation).

Loss function for a regularized GLM: \[-l(\beta\mid x,y)+\lambda R(\beta)\]

Types of Penalty

As in linear models, the penalty can be \(L_1\), \(L_2\), or a combination:

  1. LASSO: \[R(\beta)=\|\beta\|_1=\sum_j|\beta_j|\]
  2. Ridge: \[R(\beta)=\|\beta\|_2=\sum_j\beta_j^2\]
  3. Elastic net: \[R(\beta)=\alpha\|\beta\|_1+ \frac{1}{2}(1-\alpha)\|\beta\|_2\]

Selecting \(\lambda\)

We choose the best \(\lambda\) (i.e., the one with the best out-of-sample predictive performance) via cross-validation. Some estimators have a built-in CV version (such as LogisticRegressionCV). If that is not the case, we must run a parameter search (one option is using GridSearchCV).

from sklearn.linear_model import LogisticRegressionCV
from sklearn.linear_model import PoissonRegressor
from sklearn.model_selection import GridSearchCV

# Logistic or multinomial regression
lasso_logit = LogisticRegressionCV(Cs=10, penalty="l1", solver='saga')
ridge_logit = LogisticRegressionCV(Cs=10, penalty="l2", solver='saga')
enet_logit = LogisticRegressionCV(Cs=10, penalty="elasticnet", l1_ratios=[0.25,0.5,0.75], solver='saga')

# Poisson regression: alpha controls Ridge penalty
# For lambda search we use GridSearchCV
pois = PoissonRegressor()
# Grid of alpha values to perform the search  
parameters = {'alpha':[0.01,0.1,1]}
# Declare the grid search: estimator + parameter(s) grid
grid = GridSearchCV(pois,parameters,cv=5)
# Train model using the grid search and 5-fold cross validation
grid.fit(X_train,y_train)

Generative classification models

Definition

 

  • Logistic regression models directly the conditional mean of the response \(P(y=1|X)\)

  • Generative models involve modeling the distribution of the predictors separately for each one of the classes (i.e., each value of the response \(y\)): \(P(X|y)\)

  • Then, Bayes theorem is applied to retrieve the conditional probability \(P(y|X)\):

    \[P(y=k|X)=\frac{\pi_k P(X|y=k)}{\sum_j \pi_j P(X|y=k)}=\frac{\pi_k f_k(X)}{\sum_j \pi_j f_j(X)}\]

  • A unit is assigned to the class \(k\) for which the posterior probability \(P(y=k|X)\) is maximum

  • Why generative models?

    • In certain circumstances (separation between classes is substantial, i.e. lack of overlap) logistic regression parameters are unstable
    • Accuracy gains when predictors are approximately normal and with small sample sizes
    • Naturally extend to more than two classes

Naive Bayes

 

The trick to estimate generative models is how to tackle the estimation of \(f_k(\cdot)\). In the case of the Naive Bayes classifier a simplifying assumption is made: within the \(k\)th class the \(p\) predictors are independent. Then

\[f_k(X)=f_{k1}(x_1) \times f_{k2}(x_2) \times \ldots f_{kp}(x_p)\] Implications

  • The problem of estimating a \(p\)-dimensional joint distribution is simplified to that of estimating marginal distributions (removing the need to estimate the joint distribution between the different predictors)

  • Though independence is hardly to sustain, Naive Bayes produces rather reasonable results
